# Phase 8: Privacy Exclusion

## Overview

Refuse to capture clipboard writes that come from password managers, or that carry the de-facto "concealed clipboard" marker types. Provide a Settings UI for the user to extend the exclusion list.

## Requirements

- **Functional:** Bundle-ID exclusion list (default + user-extensible). Pasteboard-type exclusion (org.nspasteboard.ConcealedType, org.nspasteboard.AutoGeneratedType, com.agilebits.onepassword). Optional auto-clear of system pasteboard ~60 s after a paste from a known password manager.
- **Non-functional:** Exclusion check runs synchronously in the capture path, < 1 ms.

## Architecture

```
ClipboardWatcher tick
   ↓
PrivacyFilter.shouldCapture(frontmostBundleID, pasteboardTypes) → Bool
   ↓                              ↓
  no → drop                       yes → continue to snapshot

Settings:
   ExclusionsSection
      ├── default list (read-only chips)
      └── user list (editable, add by drag-drop app from Finder OR enter bundle ID)
```

### Default exclusion list (bundle IDs)

- `com.agilebits.onepassword7` · `com.agilebits.onepassword4` · `com.1password.1password` · `com.1password.1password-launcher`
- `com.apple.keychainaccess`
- `com.bitwarden.desktop`
- `org.keepassxc.keepassxc`
- `com.lastpass.LastPass`
- `com.dashlane.Dashlane`

### Concealed pasteboard markers

Recognised types that signal "do not record":

- `org.nspasteboard.ConcealedType`
- `org.nspasteboard.AutoGeneratedType`
- `com.agilebits.onepassword` (1Password's own private type)
- `Pasteboard generator type` (legacy)

## Related Code Files

- Create: `Clipstash/Privacy/PrivacyFilter.swift`
- Create: `Clipstash/Privacy/ExclusionList.swift` — defaults + user-defined, persisted in `UserDefaults`
- Modify: `Clipstash/Capture/ClipboardWatcher.swift` — call filter before snapshot
- Modify: `Clipstash/UI/SettingsWindow.swift` — Exclusions section (drag-drop, list, remove)
- Modify: `Clipstash/Paste/PasteEngine.swift` — if pasting from password-manager-originated item, schedule auto-clear

## Implementation Steps

1. **`ExclusionList`:**
   - `defaultBundleIDs: Set<String>` (the list above).
   - `userBundleIDs: Set<String>` — persisted in `UserDefaults` under `clipstash.exclusions.user`.
   - `concealedTypes: Set<String>` — fixed list above.
   - Helpers `add(bundleID:)`, `remove(bundleID:)`, `contains(bundleID:)`, `contains(type:)`.
2. **`PrivacyFilter.shouldCapture(frontmostBundleID, types) -> Bool`:**
   - If `frontmostBundleID` in default ∪ user lists → false.
   - If any pasteboard type matches `concealedTypes` (case-insensitive) → false.
   - Else true.
3. **Hook into `ClipboardWatcher` tick:** read `NSPasteboard.general.types` (cheap, no content) and the cached `frontmostApplication?.bundleIdentifier`. Call `PrivacyFilter.shouldCapture`. If false → bump `lastChangeCount`, do not read content, do not emit.
4. **Settings → Exclusions section:**
   - Two lists: "Default (built-in)" — read-only chips. "Your additions" — editable list.
   - "Add app…" button opens `NSOpenPanel` filtered to `.application` UTType; reads `Bundle(url:).bundleIdentifier`, adds to user list.
   - Drag-drop an app from Finder onto the list also adds it.
   - "Remove" per-row deletes from user list (default list cannot be edited).
5. **Auto-clear after password paste (optional, default ON):** in `PasteEngine`, when pasting an item whose `source_bundle_id` is in default exclusion list (i.e., captured *before* exclusion was added — legacy data), schedule `NSPasteboard.general.clearContents()` after 60 s. Confirm by checking `changeCount` is unchanged (user hasn't copied something new in the meantime).
6. **Telemetry-free verification:** add a debug-only counter "items dropped by filter" visible only in DEBUG builds — used in tests, never shipped UI.

## Success Criteria

- [ ] Copying from 1Password produces zero captured items.
- [ ] Copying from Keychain Access produces zero captured items.
- [ ] Adding a custom bundle ID via Settings immediately blocks captures from that app.
- [ ] Pasting `org.nspasteboard.ConcealedType` content from any app is not captured.
- [ ] Auto-clear toggle off in Settings disables the 60 s clear.

## Risk Assessment

- **Risk:** Bundle ID of a password manager changes across versions, breaking exclusion. **Mitigation:** ship the union of known historical IDs; auto-extend when a new ID writes a `ConcealedType`.
- **Risk:** False positive blocks legitimate captures from an excluded app the user actually wants. **Mitigation:** show a non-intrusive HUD "Capture skipped (excluded app)" for first N occurrences per session so the user knows the filter fired.
- **Risk:** User assumes everything is encrypted and stores secrets in pinned slots manually. **Mitigation:** README + first-run note clearly state "Clipstash stores history unencrypted at rest; do not pin secrets".
